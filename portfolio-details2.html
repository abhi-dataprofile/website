Customer Spending Classification

Project Description

This project presents a comprehensive Book Classification System aimed at providing companies with actionable insights based on customer spending patterns. The system classifies customers into two categories:

  • High-Spending Customers (Score: 1) – Spending score greater than 50.
  • Low-Spending Customers (Score: 0) – Spending score below 50.
The solution covers the full lifecycle of machine learning, from data ingestion and model training to deployment, and leverages powerful technologies to ensure scalability and collaboration.

Key Features and Technologies Used:

  • Machine Learning and Model Training: Built a predictive model using Python to classify customers based on spending habits.

    Note: The dataset used for training was relatively small, leading to challenges of overfitting in most models. This project aimed to explore approaches for handling limited datasets and improving classification performance, using techniques such as regularization, cross-validation, and feature selection.

  • Interactive Frontend with Streamlit: Developed an interactive user interface using Streamlit for real-time predictions. Access the app here: Streamlit Web App.
  • API Development with FastAPI: Created a RESTful API with FastAPI to enable external systems to access model predictions.
  • Model Tracking with MLflow: Implemented MLflow to log experiment results, manage versions, and track performance metrics.
  • Data Collaboration with Dagshub: Used Dagshub for dataset version control, enabling seamless collaboration and Git-based dataset tracking.
  • Containerization with Docker: Dockerized the entire application to ensure consistent deployment across environments, with separate Docker images for:
    • Streamlit frontend
    • FastAPI backend
    • MLflow tracking server
  • Cloud Deployment on AWS: Deployed on AWS for high availability and scalability, using Nginx as a reverse proxy for load balancing and optimized routing.
