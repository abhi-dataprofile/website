Data Migration Project from Salesforce to Jira

Project Overview:
Objective: Migrate all claims, returns, and ticket data from Salesforce to Jira to reduce costs.
Cost Impact: Estimated annual savings of $500k.
Stakeholders: Senior Manager (supervised the project) and cross-functional teams.
Role: Project lead responsible for planning, execution, and ensuring timely delivery.

Key Steps and Execution:

  1. Initial Planning and Understanding Requirements:
    • Met with the Senior Manager to align on goals and timelines.
    • Engaged with the Salesforce team to gather access, documentation, and workflow insights.
    • Created a Proof of Concept (POC) to secure feasibility and buy-in.
  2. Preparation and Analysis:
    • Reviewed Jira documentation to understand API structure and field requirements.
    • Mapped Salesforce fields to Jira, creating custom fields as needed.
  3. Data Extraction and Transformation:
    • Extracted data from Salesforce while ensuring data integrity.
    • Addressed field mismatches using manual and programmatic approaches.
    • Generated JSON payloads for each team’s tickets.
  4. Data Migration and API Integration:
    • Used POST requests to push JSON data to Jira.
    • Migrated data team by team to minimize disruptions.
    • Conducted unit testing for each batch to ensure data accuracy.
  5. Quality Assurance and Testing:
    • Collaborated with testers to validate the data.
    • Addressed identified issues to ensure data completeness and accuracy.
    • Automated the process using cron jobs for regular updates.
  6. Post-Migration Validation and Continuous Improvement:
    • Notified teams to verify data in Jira.
    • Resolved feedback-related issues, including requests for additional fields.
    • Provided training to ensure smooth adoption of Jira.

Challenges and Solutions:

  • Challenge: Some Salesforce fields were redundant but later deemed necessary by the teams.
    Solution: Dynamically updated the mapping and handled new custom fields as needed.
  • Challenge: Time constraints and high visibility of the project.
    Solution: Created a detailed project plan with milestones and early testing to avoid delays.

Outcome:

- Successfully migrated all data from Salesforce to Jira, reducing costs by $500k annually.
- Ensured minimal downtime and data consistency.
- Automated data updates using cron jobs, improving maintainability.
- Received positive feedback from all teams regarding Jira’s usability.
