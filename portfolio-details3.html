LLM chatbot

LLM Chatbot for Buffalo Center for Arts and Technology (BCAT)

Objective:

To develop a conversational chatbot using Snowflake tables, OpenAI API, and Streamlit that retrieves data from Snowflake based on user queries and showcases the results interactively. The goal is to enhance information accessibility for stakeholders by allowing natural language queries to generate actionable insights.

Project Description:

This project leverages a Large Language Model (LLM) to act as an interface between stakeholders and the data stored in Snowflake. The chatbot is built using Streamlit for user interaction, OpenAI's API for natural language understanding and query generation, and Snowflake for secure data storage and retrieval.

Workflow Overview:

  1. Knowledge Base as Prompts: Used structured prompts to guide the LLM in query generation and established key rules for query optimization and limiting access to sensitive information.
  2. Data Source and Query Generation: Provided the LLM with metadata about Snowflake tables (e.g., columns, data types) and incorporated stakeholder-specific query formats to tailor responses to business needs.
  3. Query Execution and Display: Sent the generated SQL query to Snowflake for execution and retrieved/displayed results within the Streamlit application in tabular or visual formats.

Key Features:

  • Dynamic SQL Generation: Automatically generates SQL queries based on stakeholder input.
  • Secure Data Access: Follows rules for accessing and querying specific Snowflake tables.
  • User-Friendly Interface: Built with Streamlit for intuitive interaction and visualization.
  • Real-Time Data Retrieval: Retrieves up-to-date information directly from Snowflake.

Skills and Technologies Used:

  • Natural Language Processing: OpenAI API
  • Data Storage and Querying: Snowflake
  • Web Application Framework: Streamlit
  • Prompt Engineering: Custom rule-based prompts for LLM
  • Version Control: GitHub (Repository Link)

System Architecture:

1. User Input (Stakeholder's Query) → 2. LLM (OpenAI API) → 3. Knowledge Base Rules →  
4. SQL Query Generation → 5. Snowflake Table Access → 6. Data Retrieved → 7. Display on Streamlit App

Key Achievements:

  • Enhanced Accessibility: Provided stakeholders with direct access to institutional data using natural language queries.
  • Query Optimization: Reduced manual query crafting time by over 50%.
  • Data Security: Incorporated predefined rules to prevent unauthorized or overly broad data access.
  • Scalability: The chatbot framework can easily integrate additional tables or data sources.

Future Improvements:

  • Real-Time Context Awareness: Add user-specific history and context for more personalized responses.
  • Dashboard Integration: Provide additional options to download reports or visualize data with more customization.
  • Enhanced Model Tuning: Fine-tune the prompts and models for more accurate and efficient SQL generation.
